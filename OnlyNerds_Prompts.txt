==============================================================
  OnlyNerds — Design Prompts & Intent Behind the UI
  A record of the creative direction that built this site.
==============================================================

This file documents the actual prompts, decisions, and instructions
that produced the OnlyNerds visual language. Written so that anyone
can reproduce the aesthetic or extend it correctly.

--------------------------------------------------------------
PART 1 — CORE PHILOSOPHY (READ THIS FIRST)
--------------------------------------------------------------

"I don't want a website. I want an artifact. Something that looks
like it was designed by a typographer who codes, not a developer
who learned Figma. Make every decision feel deliberate."

"If it looks like a Tailwind starter template, start over.
The fastest way to identify AI-generated UI is the Z-pattern layout
(image-left / text-right alternating), the same 8px border-radius
on everything, and the use of system blue as the accent color.
Eliminate all three immediately."

"OnlyNerds should feel like a well-funded independent magazine
built by engineers. Think Stripe Docs meets A24 Films meets
a physical job board pinned to a concrete wall."


--------------------------------------------------------------
PART 2 — COLOR SYSTEM
--------------------------------------------------------------

"Replace every standard web color with a sophisticated neutral
palette that has warmth baked in. Avoid pure neutral grays —
they look sterile and corporate. Every gray should lean
slightly warm, as if printed on aged paper."

Exact values used in OnlyNerds:
  Background deep:     #0D0D0D   (near black, warm undertone)
  Card surface:        #141410   (slightly lighter, still very dark)
  Border default:      #1C1C18   (barely visible, warm dark)
  Border hover/active: #323028   (warm charcoal)
  Text primary:        #EDEDEA   (off-white, not pure white)
  Text secondary:      #A09E96   (warm mid-gray)
  Text muted:          #706E66   (warm dim gray)
  Text faint:          #4A4844   (barely legible, for placeholder text)
  Selection:           rgba(160, 158, 150, 0.2)  (warm gray highlight)

"No pure white (#FFFFFF). No pure black (#000000). No system blue.
No standard green or red for status. If you need a success indicator,
use a filled warm circle. If you need an error state, use a hollow
stroke circle. Color should never carry meaning alone."

Sector accent colors (muted, never saturated):
  H1B / Growth:        #7AB87A   (muted sage green)
  Private Markets:     #C49A3A   (aged amber, like old brass)
  VC / Portfolios:     #9B8EC4   (dusty violet)
  Resources / Info:    #6B9AC4   (slate blue, desaturated)
  Orange cursor ring:  #fb923c   (the one bright element — intentional contrast)

"The orange cursor ring is the only saturated color on the entire
site. This is intentional. It draws the eye to cursor interaction
without cluttering the content palette."


--------------------------------------------------------------
PART 3 — TYPOGRAPHY SYSTEM (THE THREE-FONT RULE)
--------------------------------------------------------------

"Use exactly three fonts. One optical serif for display headings.
One humanist sans for body text and UI. One monospace for every
technical element. Never deviate. Never add a fourth."

Font assignments:
  Fraunces         — H1 display headings only
  Space Grotesk    — body text, h2, h3, paragraph content
  JetBrains Mono   — nav links, buttons, labels, stats, code, inputs

"The combination of Fraunces (a high-contrast variable optical serif
with a softer 'wonky' character) with JetBrains Mono (a developer
tool font) creates a tension between editorial warmth and technical
precision. That tension IS the personality of the site."

Heading style for H1:
  font-family: Fraunces, Georgia, serif
  font-weight: 800-900
  letter-spacing: -0.025em
  line-height: 0.92 to 0.95  (tighter than default — intentional overlap)
  font-size: clamp(2.8rem, 8vw, 5.5rem)  (fluid, never fixed)

"The H1 line-height should be tight enough to feel like the words
are pressing against each other. 0.92 is the target. This is an
editorial technique from print magazines — not standard web spacing."

"Use a lighter italic weight for the subtitle inside the H1 block.
This creates a two-tone heading that feels designed, not generated."

Example:
  Career Intelligence          <- 900 weight, normal
  for Serious Builders         <- 300 weight, italic, text-slate-400

Body text rule:
  Use font-mono (JetBrains Mono) for ALL descriptive body text,
  stats, and supporting copy. Only use Space Grotesk for headings
  and major UI sections. This gives the site a 'built-by-hand' look.

Navigation labels:
  font-family: JetBrains Mono
  font-size: 11px
  text-transform: uppercase
  letter-spacing: 0.13em
  color: #706E66 (muted), #EDEDEA (active)

Section labels / eyebrows:
  font-family: JetBrains Mono
  font-size: 10px
  text-transform: uppercase
  letter-spacing: 0.22em
  color: #706E66
  prefix with "// " to reference developer comment syntax


--------------------------------------------------------------
PART 4 — LAYOUT AND SPACING
--------------------------------------------------------------

"Asymmetrical spacing is the single biggest signal of intentional
design versus template design. The hero section must have massive
top padding to create upward momentum. Tight bottom padding so the
content wants to spill into the next section."

Hero section spacing:
  padding-top: 160px
  padding-bottom: 40px

"Never use the same padding top and bottom on a hero. It looks like
a Bootstrap component. The asymmetry communicates editorial control."

"Avoid the Z-pattern layout (image-text alternating rows). It is
the single most identifiable pattern of AI-generated landing pages.
Instead use a 12-column grid with an 8/4 or 9/3 split. Put the
primary content in the larger column and use the smaller column
for a vertical stats rail, not an illustration."

Hero grid structure:
  Left column:  8/12 — headline, subtext, CTA buttons
  Right column: 4/12 — vertical stat stack (H1B count, firm count, etc.)
                        separated by a left border-line, content anchored
                        to the bottom of the column

Stats on the right rail (desktop):
  Each stat is stacked vertically, separated by a 0.5px border-bottom
  Big number in Fraunces, label in JetBrains Mono uppercase small

"The right stat column acts as a sidebar. It should sit quietly,
not compete with the headline. Let the large serif numerals carry
the weight. No circles, no icons, no backgrounds."

Tile grid technique:
  <div class="grid grid-cols-3 gap-px bg-slate-800">
    <div class="bg-slate-900 p-7">...</div>
  </div>

"Instead of adding borders to individual tiles, set the parent grid
to have a very dark background (#323028 or #1C1C18) and set
gap-px (1px gap). This creates razor-sharp 1px dividers between
tiles without any border markup. It scales to any grid size and
always looks precise. Standard borders add up to 2px — this is 1px."


--------------------------------------------------------------
PART 5 — CARD AND TILE RULES
--------------------------------------------------------------

"Zero border-radius on ALL containers. This is the non-negotiable
rule of the design system. Not 2px, not 4px. Zero. Sharp corners
signal brutalism, craft, and intent. Rounded corners signal a
Tailwind components library was opened and defaults were kept."

Card definition:
  background: #141410
  border: 0.5px solid #1C1C18
  border-radius: 0  (hard requirement)
  border-radius exception: rounded-full is allowed for dot indicators only

Card hover behavior:
  background: #1C1C18
  transform: scale(1.015)   (barely perceptible — not a bounce, a breath)
  transition: 0.5s cubic-bezier(0.23, 1, 0.32, 1)
  z-index: 1 with isolation: isolate (prevents overlapping tile edges)

"The 0.5 second cubic-bezier hover is the 'lazy luxury' feel.
It should feel like the card is confidently settling, not snapping.
Use this exact easing function: cubic-bezier(0.23, 1, 0.32, 1).
It is the same curve used in iOS spring animations."

"scale(1.015) is intentionally small. If you go to 1.03 or 1.05,
it starts to feel like a children's website. 1.015 is adult. The
card acknowledges you noticed it, that is all."

"No box shadows. Not on cards, not on modals, not on anything.
Shadows are a skeuomorphic relic. Borders at 0.5px opacity do the
same work without the blur and blob effects."

"All borders: 0.5px solid. Not 1px. The half-pixel renders as a
sub-pixel line on Retina/HiDPI displays and looks like a hairline
engraving. On non-Retina screens it still looks thin. Go thinner
than any competitor's border."


--------------------------------------------------------------
PART 6 — FILM GRAIN OVERLAY
--------------------------------------------------------------

"Apply a persistent film grain noise overlay across the entire
viewport. It must sit above all content using z-index: 9998 but
below the cursor at 99999. It gives the UI tactile texture —
as if the screen has physical presence, like a printed piece."

Implementation (in global.css using body::after pseudoelement):
  content: ''
  position: fixed
  inset: 0
  width: 100vw, height: 100vh
  pointer-events: none
  z-index: 9998
  opacity: 0.028  (2.8% — barely visible, but you notice when it's gone)
  background-image: SVG inline with feTurbulence fractalNoise
    baseFrequency: 0.85
    numOctaves: 4
    stitchTiles: stitch
  background-size: 256px 256px
  background-repeat: repeat

"The feTurbulence SVG approach is the correct one. Do not use a
PNG noise texture — it increases page weight by 50-200KB and
doesn't scale. The inline SVG tile is ~300 bytes and infinitely
scales to any resolution. The stitchTiles='stitch' attribute
prevents visible seams at tile edges."

"0.028 opacity is the calibrated value. 0.02 is barely there.
0.05 is distracting. 0.028 sits in the sweet spot where it
reads as texture not noise."


--------------------------------------------------------------
PART 7 — DUST PARTICLES BACKGROUND
--------------------------------------------------------------

"Add a canvas-based ambient particle system to the background.
Not a hero animation, not a cursor follower — just 90 small
dust particles floating across the full viewport at all times.
They should feel like particles caught in still air, not a
screensaver."

DustParticles.astro — key parameters:
  quantity: 90 (on most pages), 140 (default if unspecified)
  particle radius: 0.8 to 2.6px (mixed sizes for depth illusion)
  particle alpha: 0.18 to 0.66 (varied opacity, some bright, some faint)
  color: rgba(245, 240, 230, alpha)  (warm white, matches the text palette)
  speed cap: 2.2px per frame
  damping: 0.97 per frame (retains energy, slight drift)
  micro-turbulence: ±0.09 per axis per frame (makes paths organic)
  kick interval: every 18-58 frames, random velocity redirect

"The kick interval is what separates this from a simple float
animation. Every 18-58 frames each particle randomly changes
direction. This makes the motion look genuinely organic — like
real suspended dust — rather than particles looping in programmed
circles. The random interval per particle means they never move
in unison."

"Pause the animation when the browser tab is hidden.
document.addEventListener('visibilitychange', ...) with
cancelAnimationFrame. This saves CPU on background tabs."

"The canvas sits at z-index: 1, behind all content but above the
body background. Pointer-events: none so it never interferes with
clicks."


--------------------------------------------------------------
PART 8 — CUSTOM CURSOR SYSTEM
--------------------------------------------------------------

"Hide the default system cursor entirely on desktop. Replace it
with a custom circle cursor that expands and glows on interactive
elements. This makes every interaction feel intentional."

CSS (in global.css):
  html { cursor: none; }
  a, button, input { cursor: none; }

The cursor element (#cursor):
  Fixed position, z-index: 99999 (above grain overlay at 9998)
  18px x 18px default size
  border: 1.5px solid rgba(251, 146, 60, 0.85)  (orange ring)
  border-radius: 50%  (exception to the no-radius rule — it's a circle)
  box-shadow: 0 0 6px rgba(251, 146, 60, 0.3)  (soft ambient glow)
  transition: all 0.25s cubic-bezier(0.23, 1, 0.32, 1)
  transform: translate(-50%, -50%)  (cursor point is center of circle)
  pointer-events: none  (cursor must never intercept its own clicks)
  will-change: left, top  (GPU optimization for smooth tracking)

Hover state (.is-hovering):
  width: 38px, height: 38px  (expands from 18px to 38px)
  background: rgba(251, 146, 60, 0.07)  (subtle fill appears)
  border-color: rgba(251, 146, 60, 1.0)  (fully opaque orange)
  box-shadow: 0 0 14px rgba(251, 146, 60, 0.45)  (stronger glow)

JavaScript tracking logic (in Layout.astro <script>):
  - Only activates if window.matchMedia('(pointer: fine)').matches
    (touch devices and tablets get hidden cursor element instead)
  - mousemove: direct style.left/top assignment (fastest, no RAF needed)
  - mouseover: check if target or closest ancestor is a/button/input/etc.
    If yes: cursor.classList.add('is-hovering')
  - mouseleave/mouseenter: fade cursor out/in (opacity 0/1)

"The pointer: fine media query is critical. On touch devices and
iPads, pointer is 'coarse', not 'fine'. Those devices use tap,
not hover. The cursor system must not activate for them — it would
just show a floating orange circle doing nothing."

"The orange ring is the only saturated color on the site. It was
chosen because it reads as warm and energetic against the near-black
background without reading as a warning or error signal (red would
be wrong, green would be wrong). Orange is the one color that says
'I see you' without saying anything else."

"The transition on the cursor itself creates the lag effect.
The circle moves with a 0.25s spring delay behind the actual mouse.
This slight lag feels premium. The circle is always chasing the
pointer, never quite at it."


--------------------------------------------------------------
PART 9 — SCROLLBAR
--------------------------------------------------------------

"Style the scrollbar to match the design system. It should be
barely visible — an almost-hidden functional element, not a
design feature."

  scrollbar width:  4px
  track background: #0D0D0D (same as page background, invisible)
  thumb background: #1C1C18 (just barely visible)
  thumb hover:      #323028
  border-radius:    0  (consistent with zero-radius rule)

"A 4px wide, near-invisible scrollbar on a dark background looks
like a ruled line on paper. It tells the user where the scroll
zone is without announcing itself."


--------------------------------------------------------------
PART 10 — TEXT SELECTION
--------------------------------------------------------------

  ::selection {
    background: rgba(160, 158, 150, 0.2);
    color: #F5F5F0;
  }

"Never use the default browser blue selection highlight. Replace it
with a warm gray semi-transparent overlay. The selected text
brightens slightly (color: #F5F5F0) as if it's been lit from
behind. It should feel like highlighting text in a physical book."


--------------------------------------------------------------
PART 11 — BUTTONS
--------------------------------------------------------------

"All buttons use JetBrains Mono, uppercase, tracked out at 0.13-0.16em.
They should feel like terminal commands, not UI controls."

Primary CTA (light):
  background: #F2F2F0 (light, high contrast against dark page)
  color: #111111
  padding: 12px 24px
  font: JetBrains Mono, 11px, uppercase, 700, tracking 0.16em
  hover: background: #FFFFFF
  no border, no shadow, no radius

Secondary CTA (ghost):
  background: transparent
  border: 0.5px solid #323028
  color: #A09E96
  padding: 12px 24px
  font: JetBrains Mono, 11px, uppercase, 700, tracking 0.16em
  hover: border-color: #706E66, color: #EDEDEA

"The primary/secondary button pairing intentionally inverts the
background relationship. On a dark site, a light button is the
highest-contrast action available. It does not need color."

"Avoid ghost buttons that use a colored border. That is a mid-2010s
startup design pattern. Use a very dark gray border that brightens
to warm gray on hover. The hover effect is the message."


--------------------------------------------------------------
PART 12 — HOVER STATES (GLOBAL TEXT BRIGHTENING)
--------------------------------------------------------------

"When a card or tile is hovered, text inside it should brighten.
This is done globally rather than per-component so it works
automatically on any new component added."

In global.css:
  [class*="group"]:hover p,
  [class*="group"]:hover li,
  [class*="group"]:hover span:not([style*="background"]) {
    color: #C8C6BE;
  }

  Headings on direct hover:
  h1:hover, h2:hover, h3:hover { color: #F5F5F0; }

"The :not([style*='background']) exclusion prevents the brightening
from overriding pill/badge colors that have inline background styles.
Without this, colored tags would have their text color overridden
by the global rule."


--------------------------------------------------------------
PART 13 — NAVIGATION BAR
--------------------------------------------------------------

"The navbar must be sticky at top. It should start fully opaque
(matching the page background) and develop a glass blur effect
only after the user scrolls past 24px. This prevents the glass
effect from obscuring content before the user has moved."

  Default:  background: #0D0D0D, no blur
  Scrolled: background: rgba(13, 13, 13, 0.92), backdrop-filter: blur(16px)
  border-bottom: 0.5px solid #1C1C18 (permanent)
  height: 96px (h-24)
  transition: background 0.4s ease, backdrop-filter 0.4s ease

Active nav link indicator:
  A 1px horizontal line at the bottom of the active link.
  Implemented as a Framer Motion <motion.span> with layoutId="nav-active-line"
  This causes the line to animate (slide) between active links on navigation.
  background: #A09E96 (warm gray, not a color accent)
  height: 1px

"The animated active indicator is a single shared element that
travels between nav items. This is the Framer Motion layoutId
pattern. When the user navigates between pages, the line
visually slides to the new active item. It is a subtle detail
that most users won't consciously notice but will feel."

Nav links (desktop):
  font: JetBrains Mono, 11px, uppercase, 500, tracking 0.13em
  default color: #706E66
  active color: #EDEDEA
  hover color: #C8C6BE

Nav links (mobile drawer):
  Full-width rows, 0.5px border-bottom between each item
  Same typography as desktop
  Framer Motion height animation: 0 to auto with opacity 0 to 1

Mobile hamburger:
  Three 1px lines (not bars, lines — 20px wide, 1px tall)
  The top and bottom lines rotate ±45 degrees to form an X on open
  The middle line fades and scales to 0
  All transitions: 0.22s cubic-bezier(0.23, 1, 0.32, 1)


--------------------------------------------------------------
PART 14 — FOOTER
--------------------------------------------------------------

"The footer is a 4-column editorial footer, not a standard sitemap
footer. It should feel like the colophon of a printed publication."

Column structure (desktop):
  30% — Large logo (SVG, clamp height 6rem to 14rem)
  23% — Pages column (links with hover slide-in line effect)
  23% — Creator column (name, role, social links as tiny bordered buttons)
  23% — Built in Association With (Alphonso AI branding block)
  1%  — Right padding spacer

Footer link hover effect:
  Each link has a hidden 0px-wide horizontal line before it.
  On hover: the line animates from 0 to 16px width (w-4).
  This creates a "sliding rule" reveal that pushes the link label.
  transition: all 0.3s

  <a class="group flex items-center gap-2">
    <span class="inline-block h-px w-0 bg-slate-400 transition-all duration-300 group-hover:w-4"></span>
    Page Name
  </a>

"The sliding rule hover is a print design reference. Magazine
footers and editorial credits use this pattern. It is the correct
footer behavior for a brutalist site."

Bottom bar:
  Tagline left, copyright + license right
  Both in JetBrains Mono 10px, uppercase, warm gray
  0.5px top border matching the footer separator


--------------------------------------------------------------
PART 15 — SECTION EYEBROWS / LABELS
--------------------------------------------------------------

"Every section needs a label above the heading that identifies
what the section is. These labels reference developer comment
syntax (// ) and use the highest tracking of anything on the site."

Format:
  // Section Name
  font: JetBrains Mono, 10px, uppercase, 500
  letter-spacing: 0.22em  (the widest tracking used anywhere)
  color: #706E66

"0.22em letter-spacing on 10px text creates maximum legibility
for tiny uppercase labels. It is wider than most web typography
goes. This extreme tracking is what makes the labels read as
designed rather than just small."


--------------------------------------------------------------
PART 16 — INPUTS
--------------------------------------------------------------

"Inputs should look like terminal prompts, not form fields.
Use the monospace font, dark background, and hairline border.
No rounded corners. No shadows."

  background: #141410
  border: 0.5px solid #323028
  border-radius: 0
  font: JetBrains Mono, 13px
  color: #EDEDEA
  placeholder-color: #706E66
  focus: border-color: #706E66, background: #1C1C18
  outline: none (replaced by border-color change)

"The placeholder color (#706E66) is the same as the muted nav
link color. This is intentional — it creates a visual hierarchy
that reads as 'this is a secondary element' without any additional
design work."


--------------------------------------------------------------
PART 17 — FRAMER MOTION PATTERNS
--------------------------------------------------------------

"Use AnimatePresence with very short durations. Animations on a
data-heavy site should be invisible to the user — they should
feel like transitions, not animations. 0.12s for card entry,
0.28s for drawer open/close."

Card list entry (staggered):
  initial: { opacity: 0, y: 8 }
  animate: { opacity: 1, y: 0 }
  transition: { duration: 0.12, delay: Math.min(i * 0.025, 0.4) }

"The delay cap at 0.4 seconds is critical. Without it, the 50th
card in a grid would have a 1.25s delay — the user would watch
cards appear for 2 seconds. Cap the maximum cumulative delay."

"y: 8 (8px upward travel on entry) is the correct value for a
data card. y: 20 or y: 40 is too theatrical. The card should
feel like it was always there and just became visible."

Drawer / accordion:
  height: 0 to 'auto' with opacity 0 to 1
  duration: 0.28s, ease: [0.23, 1, 0.32, 1]

"Never animate to a fixed height when 'auto' will work.
Framer Motion handles height: 'auto' animations correctly
without needing to measure the DOM."


--------------------------------------------------------------
PART 18 — CONSTRAINTS (THE DON'T LIST)
--------------------------------------------------------------

These are the absolute constraints that define the design system.
Violating them produces work that looks like a template.

  NO gradients — zero, anywhere, ever
  NO glow effects — the cursor ring is the only exception
  NO rounded corners — zero border-radius except rounded-full on dots/circles
  NO box shadows — hairline borders replace all shadow use cases
  NO pure black (#000000) — use #0D0D0D minimum
  NO pure white (#FFFFFF) — use #F5F5F0 maximum
  NO system blue — it signals 'default browser style'
  NO colored borders on ghost buttons — only warm gray
  NO em-dashes in user-visible strings — use hyphens
  NO Z-pattern image/text alternating layout
  NO four or more fonts
  NO CSS animations on content unless intentional (use Framer Motion)
  NO client:only in Astro — always client:load for SSR hydration
  NO gradient backgrounds behind text — grain overlay only


--------------------------------------------------------------
PART 19 — THE ONE INTENTIONAL RULE-BREAK
--------------------------------------------------------------

"Every design system needs one intentional violation to feel human.
An aesthetic that follows all its own rules too perfectly looks
machine-made. OnlyNerds breaks exactly one rule."

The orange cursor ring (#fb923c) is the only saturated, non-neutral
color on the entire site. It is the only element with a glow.
The rest of the site is desaturated warm grays and near-black.

"The cursor ring works because it is isolated. If there were
three saturated colors, the orange would be noise. With only one,
it becomes a signature. Every site built on this system should
pick exactly one rule to break — and commit to it completely."


--------------------------------------------------------------
PART 20 — REPRODUCTION NOTES
--------------------------------------------------------------

If you want to build something with this visual language:

1. Start with background #0D0D0D, immediately add the grain overlay.
   Everything looks better with grain. It is the cheapest improvement.

2. Set border-radius: 0 globally, then add back rounded-full
   for any intentional circles only.

3. Install Fraunces + Space Grotesk + JetBrains Mono.
   Assign them to h1, body, and everything technical respectively.
   Do not deviate.

4. Use gap-px on a dark-background grid parent instead of borders
   on individual tiles.

5. Set all card hover transitions to 0.5s cubic-bezier(0.23, 1, 0.32, 1)
   with scale(1.015). This is the one transition that most distinguishes
   this aesthetic from standard Tailwind components.

6. Add the cursor ring last. Once it is in, the site feels finished.

7. Never add a gradient. Never add a shadow. Never add a fourth font.
   These are load-bearing constraints, not preferences.
